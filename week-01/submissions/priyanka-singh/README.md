## What I Learned
Today I learned how AWS IAM securely controls access to AWS resources.

I understood the difference between the Root User and IAM Users, how Groups simplify permission management, and how Policies define what actions a user can perform.

One important observation during the lab was that AWS follows a **default deny** model. I tested this by creating a user with **AmazonEC2ReadOnlyAccess** and found that the user could not access or create S3 buckets because no S3 permissions were granted. This helped me understand that AWS only allows actions that are explicitly permitted by the attached policy.

I also learned the Principle of Least Privilege by giving a user only read access to an S3 bucket and verifying that upload and delete operations were denied.

## Key Takeaway
AWS follows the Principle of Least Privilege and a default deny model. A user can perform only the actions that are explicitly allowed by the attached IAM policy. Granting only the minimum required permissions makes AWS environments more secure.
